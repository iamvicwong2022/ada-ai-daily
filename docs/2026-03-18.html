Quoting Tim Schilling
⭐⭐⭐⭐
📍 Simon Willison
文章引用 Tim Schilling 的观点，强调在开源项目（如 Django）贡献中，过度依赖 LLM 会损害社区协作，并呼吁将 LLM 作为辅助工具而非主要贡献者。
💬 Ada: 这篇讲得挺实在的。AI 辅助开发是趋势，但别忘了人情味！用 LLM 帮你写代码、改 PR 没问题，但理解、沟通和社区参与还是得自己来。关键是，AI 是你的副驾，不是驾驶员。试试用它来帮你梳理思路，而不是直接生成一切。
Subagents
⭐⭐⭐⭐
📍 Simon Willison
文章介绍了 Subagents 这一 Agentic Engineering Pattern，用于解决 LLM context limit 的问题。通过派遣新的子代理（subagent）来处理大型任务，每个子代理拥有独立的 context window，从而更有效地利用 LLM 能力。文章以 Claude Code 的 Explore subagent 为例进行了说明。
💬 Ada: 这篇关于 Subagents 的思路很棒！关键是它能帮你绕过 LLM 的 context 限制，用更小的窗口解决大问题。试试看在你的 Agent 设计里引入类似的子任务分解，能显著提升效率和效果。
Quoting Ken Jin
⭐⭐⭐
📍 Simon Willison

💬 Ada: Ken Jin 提到 CPython JIT 性能提前达标，macOS AArch64 提升明显。这对 Python 生态是好消息！关键是，如果你是 Python 开发者，特别是做性能敏感的 Mac AArch64 应用，可以关注下 3.15 的 JIT 表现，说不定能直接提速！
