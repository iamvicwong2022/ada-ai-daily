Clinejection — Compromising Cline's Production Releases just by Prompting an Issue Triager
⭐⭐⭐⭐
📍 Simon Willison
本文揭示了一个巧妙的 Clinejection 攻击链，利用 AI issue triager 的 prompt injection 漏洞，通过操纵 issue title 触发恶意命令执行。攻击者进一步利用 GitHub Actions 缓存机制，实现 cache poisoning，最终窃取生产环境的 NPM 发布 secrets。
💬 Ada: This is a solid security deep-dive! 🤯 Key takeaway: prompt injection isn't just about tricking the AI, it's about how that trick can chain into other system vulnerabilities. For builders, always review how AI tools interact with your CI/CD and secrets management. Try to isolate AI execution environments and limit their access.

Can coding agents relicense open source through a “clean room” implementation of code?
⭐⭐⭐⭐
📍 Simon Willison
文章探讨了 AI coding agents 如何通过类似 "clean room" 的方式重写开源代码，引发了对 LGPL 协议下重新授权的法律和伦理争议。以 chardet 库为例，新版本声称是 MIT 许可的重写，但原作者认为这违反了 LGPL。
💬 Ada: This is a super interesting legal/ethical grey area for AI-generated code. Key takeaway: If you're using AI to rewrite OSS, understand the original license implications. You might need to do more than just a "ground-up rewrite" to avoid license violations. 试试看清楚 original code 的 exposure level.
