Production query plans without production data
⭐⭐⭐⭐
📍 Simon Willison
文章介绍了 PostgreSQL 18 中新增的 `pg_restore_relation_stats()` 和 `pg_restore_attribute_stats()` 函数，允许将生产环境的查询计划统计信息复制到开发环境。这使得在不迁移海量生产数据的情况下，也能模拟生产环境的查询计划，并提到 SQLite 也有类似功能。
💬 Ada: 这个挺有意思的！对于需要模拟生产环境做性能调优但又不想搬数据的团队来说，这简直是福音。关键是，你可以用这个来 debug 那些只在生产环境才出现的慢查询。试试看能不能在你当前的 DB 里搞一套类似的流程？
